3. Детали по платформам

Vertex AI: Google активно развивает парадигму агентов через Gemini Live и внедряет AI прямо в уровень работы с данными (SQL). Упор сделан на Enterprise-сектор и цифровую медицину.

Сигналы: LLM-powered SQL функции, What’s new with Google Data Cloud

AWS Bedrock: AWS сфокусировались на безопасности агентов, внедряя управление через Chrome-политики и поддержку MCP для Cisco-решений. Также представлены новые возможности для real-time голосовых агентов.

Сигналы: Security for MCP, Securing AI agents

Azure Foundry: Платформа достигла стабильности в ряде компонентов, включая Foundry Local, и обновила стек поддерживаемых моделей. Фокус сместился на инструменты разработки и визуализации для AI.

Сигналы: Foundry Local 1.1 GA, Foundry Local 1.1: Live Transcription

AutoGen: В данный период обновлений не зафиксировано.

Сигналы: Нет данных. GitHub

AgentForce: Salesforce совершила прорыв в сторону Developer Experience, внедрив версионируемый метаданный и human-readable скрипты. Также улучшена визуализация отладки через AgentLens.

Сигналы: New Metadata Lifecycle, Agentforce Metadata

LangGraph: Произошел переход от альфа-версий к официальным релизам всей экосистемы. Улучшена стабильность чекпоинтов для SQL-хранилищ.

Сигналы: Стабильный релиз 1.2.0, LangGraph Releases

n8n: Получение инвестиций от SAP и нативная интеграция в их экосистему подтверждают зрелость n8n как Enterprise-инструмента. Платформа продолжает укреплять позиции в крупном корпоративном секторе.

Сигналы: Инвестиция $5.2bn от SAP, n8n и SAP

5. Выводы для команды

  1. Конкуренты начали массово копировать паттерны CI/CD и Git-native разработки (Salesforce), что сокращает наше технологическое преимущество в простоте деплоя. Необходимо ускорить развитие инструментов визуализации потоков, похожих на AgentLens.
  2. Интеграция n8n в SAP доказывает, что enterprise-клиенты ищут платформы, которые уже являются частью их экосистемы; нужно сфокусироваться на коннекторах для таких же систем.
  3. Безопасность (MCP) становится общим стандартом для всех крупных облаков — следует убедиться, что SL2 поддерживает последние спецификации MCP для сохранения совместимости.
