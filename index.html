3. Детали по платформам

Vertex AI: Google вывел в GA Gemini Enterprise Agent Platform с remote MCP-серверами и детерминированными conversational flows (DP). Это усиливает критерии Unified API + контракт (через MCP-совместимость) и Prod-ops (BigQuery AI.AGG для аналитики) (CP). Для SL2 AI Platform это давление по двум фронтам: Vertex AI приближается к нашему уровню детерминированности и observability, но пока не поддерживает self-host. Затрагивает GAP G2 (A2A-несовместимость) и G3 (нет inter-agent tracing), так как Google активно продвигает MCP как стандарт (SP). Однако их решение остаётся managed-only, что сохраняет наше преимущество для регулируемых клиентов.

Сигналы: GA Gemini Enterprise Agent Platform, BigQuery AI.AGG

AWS Bedrock: AWS выпустил в GA AgentCore Runtime с изоляцией агентов в microVM и AG-UI протокол для generative UI (DP). Это закрывает критерии Indep. deploy/scale и Secrets/tenant — теперь каждый агент Bedrock живёт в отдельной microVM с собственными ресурсами и изоляцией секретов (CP). Для SL2 AI Platform это прямое давление: managed-конкурент догоняет нас по изоляции, которая была нашим ключевым преимуществом. Затрагивает GAP G3 (нет inter-agent tracing) и G4 (нет cost tracking per agent), так как Bedrock теперь предоставляет это "из коробки" (SP). Однако Bedrock остаётся closed SaaS, что сохраняет наше преимущество для self-host клиентов.

Сигналы: GA AgentCore Runtime, microVM isolation

AutoGen: Сообщество активно обсуждает security и audit trails для multi-agent систем, включая Ed25519-аутентификацию агентов и open-source security тесты (DP). Это усиливает критерий Secrets/tenant (через криптографическую аутентификацию) и Prod-ops (через security тесты) (CP). Для SL2 AI Platform это сигнал о растущем спросе на enterprise-grade security в multi-agent системах, где мы уже имеем преимущество (AES-256-GCM, иерархия секретов). Однако AutoGen движется быстрее в community-driven security, что может стать угрозой для нашего GAP G7 (молодой проект, нет community) (SP). Отслеживаем, появятся ли у них стандартизированные решения для audit trails.

Сигналы: Ed25519 auth, security tests

AgentForce: Salesforce выпустил детерминированные conversational flows и continuous planning systems для AgentForce (DP). Это усиливает критерий Multi-tenancy (через org-level ACL) и приближает их к enterprise-grade workflow orchestration (CP). Для SL2 AI Platform это давление на наше преимущество в детерминированности и git-native dev, так как AgentForce теперь предлагает аналогичные возможности "из коробки" для Salesforce-клиентов. Затрагивает GAP G1 (нет visual builder) и G7 (нет marketplace агентов), так как Salesforce имеет готовые интеграции с CRM (SP). Однако их решение остаётся SaaS-only и привязано к Salesforce-экосистеме.

Сигналы: Deterministic flows, Continuous planning

n8n: Релиз self-improving agent skills и AI maturity framework (DP). Это усиливает критерий Code+No-code (через self-improving skills) и приближает n8n к enterprise-grade AI orchestration (CP). Для SL2 AI Platform это сигнал о растущей конкуренции в no-code сегменте, где мы имеем GAP G1 (нет visual builder). Однако n8n остаётся workflow-ориентированным инструментом, а не полноценной agent runtime платформой, что сохраняет наше преимущество в enterprise-сегменте (SP).

Сигналы: Self-improving skills, AI maturity framework
