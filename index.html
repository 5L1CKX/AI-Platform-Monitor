3. Детали по платформам

Vertex AI: Google выпустил Agent Development Kit (ADK) для создания полиморфных multi-agent систем с динамической маршрутизацией и масштабированием (DP). Это усиливает критерии Indep. scale (горизонтальное масштабирование агентов) и LLM routing (интеллектуальная диспетчеризация запросов), а также приближает Vertex к Unified API через ADK-контракты (CP). Для SL2 AI Platform это давление на G7 (зрелость) и G8 (архитектура), так как ADK предлагает enterprise-ready решение "из коробки", в то время как SL2 AI Platform требует ручной настройки оркестрации. Однако ADK остаётся привязанным к Google Cloud, что сохраняет преимущество SL2 AI Platform в self-host для регулируемых клиентов.

Сигналы: Beyond Static Prompts: ADK, Scaling LLM Inference

AWS Bedrock: AWS представил serverless A2A-gateway для discovery и маршрутизации агентов, а также AgentCore Memory с метаданными для изоляции и фильтрации данных между арендаторами (DP). Это закрывает пробелы в Multi-tenancy (изоляция памяти) и Unified API (стандартизированный A2A-контракт), а также усиливает Secrets/tenant через микро-VM изоляцию (CP). Для SL2 AI Platform это прямая угроза по G2 (lock-in в Agent Protocol) и G4 (cost tracking), так как Bedrock теперь предлагает managed-решение с прозрачным ценообразованием, в то время как SL2 AI Platform требует ручной интеграции FinOps-инструментов.

Сигналы: Serverless A2A Gateway, AgentCore Memory

AutoGen: Сообщество активно обсуждает security-стандарты для multi-agent систем, включая Ed25519-аутентификацию агентов (proveyouragent) и AlgoVoi — открытый фреймворк для верификации решений агентов (DP). Это усиливает критерий Secrets/tenant (децентрализованная аутентификация) и поднимает планку для Prod-ops (security-тесты) (CP). Для SL2 AI Platform это давление на G3 (observability) и G6 (LLM gateway), так как AutoGen предлагает готовые решения для безопасности, в то время как SL2 AI Platform требует ручной интеграции инструментов вроде OpenTelemetry.

Сигналы: Agent Authentication, AlgoVoi Decision Chain

AgentForce: Salesforce выпустил MCP-серверы с security-контролями (аутентификация, авторизация, логирование) и LWC State Managers для реактивного управления состоянием агентов (DP). Это усиливает Code+No-code (реактивные компоненты) и Secrets/tenant (изоляция MCP-серверов) (CP). Для SL2 AI Platform это давление на G1 (visual builder), так как AgentForce предлагает низкоуровневые инструменты для разработчиков, в то время как SL2 AI Platform полагается на YAML-компромисс.

Сигналы: MCP Server Security, LWC State Managers

LangGraph: Выпущены патчи безопасности (1.2.7) и интеграция с OpenRouter для LLM-routing (DP). Это усиливает LLM routing (мультипровайдерность) (CP). Для SL2 AI Platform это давление на G6 (LLM gateway), так как LangGraph предлагает зрелую обвязку для роутинга, в то время как SL2 AI Platform поддерживает собственную реализацию.

Сигналы: langgraph==1.2.7, langchain-openrouter==0.2.5

n8n: Опубликован MCP Server Security Framework и agentic AI design patterns для production (DP). Это усиливает Prod-ops (наблюдаемость и безопасность MCP-серверов) (CP). Для SL2 AI Platform это давление на G3 (observability) и G5 (data-residency), так как n8n предлагает готовые решения для безопасности, в то время как SL2 AI Platform требует ручной настройки.

Сигналы: MCP Server Security, Agentic AI Design Patterns

NetBox: Выпущен HIPAA Security Rule Compliance Pack для валидации инфраструктурных агентов и Asset Lifecycle GA для управления закупками (DP). Это усиливает Prod-ops (compliance) и Secrets/tenant (HIPAA-изоляция) (CP). Для SL2 AI Platform это давление на G5 (data-residency), так как NetBox предлагает self-hosted решение для compliance, в то время как SL2 AI Platform полагается на внешние SaaS-инструменты.

Сигналы: HIPAA Compliance Pack, Asset Lifecycle GA
