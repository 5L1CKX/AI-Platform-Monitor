3. Детали по платформам

Vertex AI: На неделе Google Cloud анонсировал ускорение inference на GKE до 92% с помощью GKE Inference Gateway (DP). Это улучшает критерий Indep. scale для Vertex AI, так как позволяет эффективнее масштабировать AI-нагрузки на Kubernetes (CP). Для SL2 AI Platform это давление на наш GAP G6 (LLM gateway), так как Google предлагает более зрелое решение для оптимизации inference, в то время как наша мультипровайдерная обвязка требует дополнительного maintenance. Однако Vertex AI остаётся managed-решением, что сохраняет наше преимущество для self-host клиентов. Сигналы: GKE Inference Gateway, Claude Fable 5. AWS Bedrock: AWS вывел в GA Bedrock AgentCore Runtime с изоляцией каждого агента в отдельной microVM и поддержкой Fully Homomorphic Encryption (FHE) для ML inference (DP). Это закрывает критерии Indep. deploy, Indep. scale и Secrets/tenant, так как агенты теперь полностью изолированы и могут обрабатывать данные в зашифрованном виде (CP). Для SL2 AI Platform это прямое давление на GAP G3 (observability) и G4 (FinOps), так как Bedrock предлагает более зрелые решения для изоляции и cost tracking. Однако Bedrock остаётся closed SaaS, что сохраняет наше преимущество для клиентов с требованиями к self-host. Сигналы: AgentCore Runtime, FHE для ML inference. Azure Foundry: Microsoft анонсировал Foundry Local для edge AI и улучшил observability для AI агентов (DP). Это улучшает критерии Self-host (за счёт локального развёртывания) и Prod-ops (за счёт observability) (CP). Для SL2 AI Platform это давление на GAP G5 (data-residency), так как Foundry теперь предлагает self-hosted observability, в то время как мы зависим от внешнего SaaS (LangSmith). Однако Foundry остаётся менее зрелым в части Git-native dev и multi-tenancy. Сигналы: Foundry Local, Observability для агентов. AutoGen: В сообществе AutoGen появились предложения по стандартизации защиты от runaway loops (SR-LoopShield) и governance interceptors для агентов (DP). Это анонсирует улучшение критериев Unified API + контракт и Secrets/tenant, так как предлагаются стандарты для безопасного взаимодействия агентов (CP). Для SL2 AI Platform это давление на GAP G2 (Agent Protocol не совместим с A2A), так как AutoGen движется в сторону стандартизации, что может усилить lock-in нашего формата. Однако предложения пока находятся на стадии обсуждения. Сигналы: SR-LoopShield, Governance interceptor. AgentForce: Salesforce анонсировал автономную квалификацию лидов в Agentforce и интеграцию с Claude Code для генерации production-ready кода (DP). Это улучшает критерии Multi-tenancy (изоляция агентов) и Code+No-code (интеграция с Claude Code) (CP). Для SL2 AI Platform это давление на GAP G1 (нет visual builder), так как Agentforce предлагает более низкий порог входа для не-разработчиков. Однако Agentforce остаётся закрытой платформой с vendor lock-in. Сигналы: Автономная квалификация лидов, Claude Code + Salesforce Skills. LangGraph: На неделе вышли минорные обновления библиотек (langchain-core, langchain-openai), без значимых архитектурных изменений (DP). Это не меняет топологию платформы (CP). Для SL2 AI Platform нет прямого давления, но LangGraph остаётся лидером в observability для LLM-вызовов, что подчёркивает наш GAP G3 (нет inter-agent tracing). Сигналы: langchain-core 1.4.3, langchain-openai 1.3.0. n8n: n8n внедрил поддержку OpenTelemetry для трассировки workflows и опубликовал гайды по LLM routing и observability (DP). Это улучшает критерии LLM routing и Prod-ops (CP). Для SL2 AI Platform это давление на GAP G3 (observability), так как n8n предлагает более зрелое решение для трассировки межагентных взаимодействий. Однако n8n остаётся инструментом для orchestration, а не полноценной платформой для агентов. Сигналы: OpenTelemetry для workflows, LLM routing.
