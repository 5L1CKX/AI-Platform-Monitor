3. Детали по платформам

Vertex AI: Google выпустил Agent Development Kit (ADK) для построения масштабируемых multi-agent систем с полиморфными агентами, что усиливает критерий Unified API + контракт за счёт стандартизации взаимодействий между агентами (DP). Это частично закрывает GAP G2 (Agent Protocol lock-in), так как ADK позволяет интеграцию с внешними стандартами через адаптеры (CP). Для SL2 AI Platform это давление на уникальность нашего git-native подхода, но ADK остаётся cloud-only, что сохраняет наше преимущество в self-host сегменте (SP). Отслеживаем, появится ли ADK в open-source или self-host варианте.

Сигналы: Beyond Static Prompts with Google ADK, Claude Apps Gateway

AWS Bedrock: AWS вывел в GA A2A-шлюз (Agent-to-Agent) с path-based routing и AgentCore Memory с метаданными для фильтрации, что усиливает Multi-tenancy и Secrets/tenant (DP). Это напрямую конкурирует с нашим Unified API + контракт и Secrets/tenant, так как Bedrock теперь предлагает аналогичную изоляцию и контроль доступа "из коробки" (CP). Для SL2 AI Platform это угроза по GAP G2 (Agent Protocol lock-in) и G4 (cost tracking), так как Bedrock интегрирован с AWS FinOps (SP). Однако Bedrock остаётся managed-сервисом, что сохраняет наше преимущество для клиентов с требованиями к self-host.

Сигналы: Serverless A2A Gateway, Structured Memory Filtering

AutoGen: В обсуждениях появились паттерны для security-развертываний, включая библиотеку proveyouragent для Ed25519-аутентификации агентов и AgentGraph для trust verification (DP). Это усиливает критерий Secrets/tenant и частично закрывает GAP G3 (observability), так как предлагает решения для межагентной безопасности и аудита (CP). Для SL2 AI Platform это сигнал к ускорению работ по G3 и G5 (self-hosted observability), так как AutoGen движется в сторону enterprise-grade security (SP).

Сигналы: Agent Authentication, AgentGraph Trust Verification

AgentForce: Salesforce выпустил GA Mobile Skills для быстрого создания мобильных приложений с агентами и опубликовал гайд по безопасности MCP-серверов (DP). Это усиливает Unified API + контракт и Secrets/tenant за счёт стандартизации MCP (Model Context Protocol) и security-контролей (CP). Для SL2 AI Platform это давление на GAP G6 (LLM gateway), так как MCP может стать альтернативой нашему кастомному LLM-роутингу (SP). Однако MCP пока не поддерживает self-host, что сохраняет наше преимущество.

Сигналы: Mobile Skills GA, MCP Server Security

n8n: Опубликованы паттерны для agentic AI, включая MCP-сервер security и визуализацию рисков (DP). Это усиливает критерий Code+No-code за счёт расширения возможностей no-code разработки (CP). Для SL2 AI Platform это сигнал к ускорению работ по GAP G1 (visual builder), так как n8n предлагает более зрелые инструменты для не-разработчиков (SP).

Сигналы: Agentic AI Design Patterns, MCP Server Security

NetBox: Выпущен HIPAA Security Rule Compliance Pack для валидации инфраструктуры и GA Asset Lifecycle для procurement (DP). Это усиливает критерий Prod-ops за счёт интеграции compliance-валидации в workflow (CP). Для SL2 AI Platform это косвенное давление на GAP G5 (self-hosted observability), так как NetBox предлагает enterprise-grade инструменты для регулируемых отраслей (SP).

Сигналы: HIPAA Compliance Pack, Asset Lifecycle GA
