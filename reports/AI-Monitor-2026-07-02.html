3. Детали по платформам

Vertex AI: На неделе Google анонсировал Agent Development Kit (ADK) для создания масштабируемых мультиагентных систем, решающий проблему "context collapse" в enterprise-окружениях (DP). Это усиливает критерий Prod-ops, так как ADK предоставляет инструменты для управления сложными workflows и динамическими бизнес-правилами (CP). Для SL2 AI Platform это давление на GAP G3 (отсутствие межагентного трейсинга) и G7 (зрелость), так как Google предлагает готовое решение для enterprise-масштабирования, в то время как SL2 AI Platform требует ручной настройки observability. Однако ADK остаётся привязанным к Google Cloud, что сохраняет преимущество SL2 AI Platform в self-hosted сценариях.

Сигналы: Beyond Static Prompts with ADK, Claude Code + Vertex AI

AWS Bedrock: AWS выпустил в GA AgentCore Runtime с изоляцией каждого агента в отдельной microVM, что закрывает критерии Indep. deploy, Indep. scale и Secrets/tenant (DP). Это напрямую конкурирует с преимуществом SL2 AI Platform в независимом масштабировании агентов (CP). Для SL2 AI Platform это угроза по GAP G3 (observability) и G4 (cost tracking), так как Bedrock теперь предлагает managed-решение с изоляцией "из коробки", в то время как SL2 AI Platform требует ручной настройки HPA и мониторинга. Однако Bedrock остаётся SaaS-платформой, что ограничивает его применение в регулируемых отраслях.

Сигналы: AgentCore Runtime GA, A2A gateway

AutoGen: В обсуждениях на GitHub появились предложения по безопасности агентов, включая библиотеку для аутентификации агентов (proveyouragent) и инструмент Scankii для защиты от prompt injection (DP). Это усиливает критерий Secrets/tenant, так как предлагает стандартизированные решения для аутентификации и контроля доступа (CP). Для SL2 AI Platform это давление на GAP G9 (секреты на уровне tenant), так как AutoGen предлагает готовые решения для безопасности, в то время как SL2 AI Platform требует ручной настройки иерархии секретов. Однако решения AutoGen пока не интегрированы в core-фреймворк и остаются сторонними инициативами.

Сигналы: Agent authentication discussion, Scankii security tool

AgentForce: Salesforce анонсировал Mobile Skills для быстрого создания мобильных приложений через простые промпты, что расширяет критерий Code+No-code (DP). Это усиливает позицию AgentForce в нише low-code решений для enterprise (CP). Для SL2 AI Platform это давление на GAP G1 (отсутствие visual builder), так как Mobile Skills снижает порог входа для не-разработчиков. Однако AgentForce остаётся привязанным к экосистеме Salesforce, что ограничивает его применение в мультивендорных окружениях.

Сигналы: Mobile Skills GA, MCP server security

n8n: Опубликованы гайды по безопасности MCP-серверов и agentic AI design patterns, включая рекомендации по аутентификации, observability и cost control (DP). Это усиливает критерий Prod-ops, так как предлагает лучшие практики для enterprise-развёртываний (CP). Для SL2 AI Platform это давление на GAP G3 (observability) и G4 (cost tracking), так как n8n предлагает готовые решения для мониторинга и управления затратами. Однако n8n остаётся визуальным оркестратором, а не полноценной agent runtime платформой.

Сигналы: MCP server security guide, Agentic AI design patterns

NetBox: Вышел релиз v4.6.4 с улучшениями UI для работы с JSON-схемами и анонсирован HIPAA-валидационный пак для проверки сетевой сегментации (DP). Это усиливает критерий Prod-ops в части compliance и observability (CP). Для SL2 AI Platform это не создаёт прямого давления, но подчёркивает важность compliance-инструментов в enterprise-окружениях. NetBox остаётся специализированным инструментом для инфраструктурного управления, а не универсальной agent-платформой.

Сигналы: NetBox v4.6.4 release, HIPAA compliance pack

Snipe-IT: В обсуждениях предложена идея модернизации UI/UX дашборда для соответствия современным стандартам (DP). Это может усилить критерий Code+No-code в части usability (CP). Для SL2 AI Platform это косвенное давление на GAP G1 (visual builder), так как улучшение usability в конкурирующих продуктах повышает ожидания пользователей. Однако Snipe-IT остаётся узкоспециализированным инструментом для IT-ассетов.

Сигналы: UI/UX modernization discussion
