3. Детали по платформам

Vertex AI: На неделе Google анонсировал Agent Development Kit (ADK) для создания масштабируемых multi-agent систем, решающий проблему "context collapse" в enterprise-окружениях (DP). Это усиливает критерии Unified API + контракт и LLM routing, так как ADK позволяет динамически адаптировать агентов к бизнес-правилам и гетерогенным данным (CP). Для SL2 AI Platform это давление на GAP G2 (Agent Protocol несовместим с A2A) и G7 (зрелость), так как Google предлагает готовое решение для enterprise-масштаба, в то время как SL2 AI Platform требует ручной настройки (SP). Однако ADK пока не поддерживает self-host, что оставляет нишу для SL2 AI Platform в регулируемых отраслях.

Сигналы: Google ADK, Claude Code + Vertex

AWS Bedrock: AWS выпустил в GA AgentCore Runtime с изоляцией агентов в отдельных microVM, что закрывает критерии Indep. deploy, Indep. scale и Secrets/tenant (DP). Это напрямую конкурирует с преимуществами SL2 AI Platform в self-hosted сегменте, так как Bedrock теперь предлагает аналогичную изоляцию "из коробки" (CP). Для SL2 AI Platform это угроза по GAP G3 (нет inter-agent tracing) и G4 (нет cost tracking per agent), так как Bedrock предоставляет эти возможности в managed-формате (SP). Однако Bedrock остаётся closed SaaS, что сохраняет преимущество SL2 AI Platform для клиентов с требованиями к data residency.

Сигналы: AgentCore Runtime, A2A gateway

AutoGen: В обсуждениях на GitHub появились предложения по безопасности агентов, включая Ed25519-аутентификацию для межагентных вызовов и AlgoVoi decision chain для верификации действий (DP). Это усиливает критерий Secrets/tenant, так как предлагает стандартизированный подход к управлению идентификацией и авторизацией агентов (CP). Для SL2 AI Platform это давление на GAP G9 (Secrets/tenant), так как текущая реализация не поддерживает криптографическую верификацию межагентных взаимодействий (SP). Однако предложенные решения пока находятся на стадии обсуждения и не интегрированы в ядро.

Сигналы: Ed25519 auth, AlgoVoi decision chain

AgentForce: Salesforce выпустил в GA Mobile Skills для быстрого создания мобильных приложений через агентов, что расширяет критерий Code+No-code (DP). Это усиливает позицию AgentForce в enterprise-сегменте, так как позволяет бизнес-пользователям создавать приложения без глубоких технических знаний (CP). Для SL2 AI Platform это давление на GAP G1 (нет visual builder), так как Mobile Skills снижает порог входа для не-разработчиков (SP). Однако AgentForce остаётся managed-решением, что ограничивает его применение в регулируемых отраслях.

Сигналы: Mobile Skills GA

n8n: Опубликованы архитектурные гайды по error handling для LLM tool calling и MCP-безопасности, что усиливает критерий Prod-ops (DP). Это улучшает наблюдаемость и надёжность multi-agent систем, приближая n8n к enterprise-требованиям (CP). Для SL2 AI Platform это давление на GAP G3 (нет inter-agent tracing) и G6 (собственная LLM gateway вместо LiteLLM), так как n8n предлагает готовые решения для этих проблем (SP). Однако n8n пока не поддерживает git-native разработку, что сохраняет преимущество SL2 AI Platform.

Сигналы: LLM error handling, MCP security

NetBox: Выпущен релиз v4.6.4 с улучшениями UI и HIPAA-валидационный пак, что усиливает критерий Prod-ops для инфраструктурных платформ (DP). Это делает NetBox более привлекательным для регулируемых отраслей, таких как здравоохранение (CP). Для SL2 AI Platform это не является прямой угрозой, так как NetBox фокусируется на инфраструктурном уровне, а не на AI-агентах (SP). Однако появление HIPAA-пака подчёркивает важность compliance-инструментов, что может быть актуально для будущих версий SL2 AI Platform.

Сигналы: v4.6.4 release, HIPAA pack

Snipe-IT: В обсуждениях на GitHub подняты темы модернизации UI и API-аутентификации (DP). Это может усилить критерий Prod-ops для asset management систем, но не затрагивает напрямую AI-платформы (CP). Для SL2 AI Platform это не является угрозой, так как Snipe-IT не конкурирует в сегменте AI-агентов (SP).

Сигналы: UI modernization, API auth
