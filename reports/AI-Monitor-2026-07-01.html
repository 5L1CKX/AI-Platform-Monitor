3. Детали по платформам

Vertex AI: Google вывел в GA Gemini Enterprise Agent Platform с полностью управляемым remote MCP-сервером, что расширяет возможности оркестрации агентов в enterprise-сегменте (DP). Это усиливает позицию Vertex AI по критерию Unified API + контракт, так как MCP-серверы обеспечивают стандартизированный интерфейс для межагентных взаимодействий, приближаясь к A2A-стандарту (CP). Для SL2 AI Platform это создаёт давление по GAP G2 (Agent Protocol не совместим с A2A), так как Google активно продвигает свой формат как de-facto стандарт. Однако Vertex AI остаётся closed SaaS, что сохраняет наше преимущество по self-host для регулируемых клиентов. Отслеживаем, когда Google выпустит A2A-совместимый адаптер.

Сигналы: GA Gemini Enterprise Agent Platform, BigQuery conversational analytics

AWS Bedrock: AWS объявил о GA AgentCore Runtime с изоляцией каждого агента в отдельной microVM, что закрывает критерии Indep. scale и Secrets/tenant (DP). Это значительное улучшение безопасности и изоляции по сравнению с предыдущими версиями, где агенты разделяли ресурсы (CP). Для SL2 AI Platform это прямое конкурентное давление, так как Bedrock теперь предлагает enterprise-grade изоляцию "из коробки", затрагивая наши GAP G3 (observability) и G4 (FinOps) — у нас нет аналогичного уровня встроенной изоляции и cost tracking per agent. Однако Bedrock остаётся managed-сервисом, что ограничивает его применение в регулируемых отраслях. Отслеживаем, появится ли у AWS per-agent cost breakdown.

Сигналы: GA AgentCore Runtime, Managed entitlements

AutoGen: Сообщество активно развивает security-инфраструктуру для multi-agent систем, включая библиотеки для agent authentication (Ed25519), security testing и trust verification (DP). Это усиливает позицию AutoGen по критерию Secrets/tenant, так как ранее агенты могли использовать hardcoded токены (CP). Для SL2 AI Platform это сигнал о растущем внимании к безопасности в multi-agent системах, что затрагивает наш GAP G3 (observability) — у нас нет встроенных механизмов для аудита межагентных взаимодействий. Однако AutoGen остаётся open-source проектом без enterprise-поддержки, что ограничивает его применение в корпоративном сегменте. Отслеживаем, появятся ли в AutoGen встроенные инструменты для distributed tracing.

Сигналы: Agent authentication, Security testing

AgentForce: Salesforce определил agentic development lifecycle для AgentForce, включая continuous planning для финансовых сервисов и deterministic conversational flows с Agent Script (DP). Это усиливает позицию AgentForce по критерию Code+No-code, так как теперь платформа предлагает визуальные инструменты для построения детерминированных workflows (CP). Для SL2 AI Platform это давление по GAP G1 (нет visual builder), так как AgentForce снижает порог входа для не-разработчиков. Однако AgentForce остаётся closed SaaS с привязкой к Salesforce-экосистеме, что ограничивает его применение вне CRM-контекста. Отслеживаем, появятся ли в AgentForce open-source компоненты.

Сигналы: Agentic development lifecycle, Deterministic flows

n8n: Выпущен фреймворк для self-improving agent skills и AI maturity framework с 5 уровнями зрелости (DP). Это усиливает позицию n8n по критерию Prod-ops, так как платформа теперь предлагает инструменты для оценки и улучшения AI-систем (CP). Для SL2 AI Platform это сигнал о растущей конкуренции в области observability и continuous improvement, что затрагивает наш GAP G3 (observability) и G7 (зрелость). Однако n8n остаётся low-code платформой с ограниченными возможностями для enterprise-разработчиков. Отслеживаем, появятся ли в n8n инструменты для distributed tracing.

Сигналы: Self-improving skills, AI maturity framework
