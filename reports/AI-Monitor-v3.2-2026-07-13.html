3. Детали по платформам

Vertex AI: На неделе Google Cloud объявил о GA AlphaEvolve для оптимизации агентных задач и представил sandbox-режим для безопасного исполнения AI-кода в Cloud Run (DP). Это усиливает критерии LLM routing (AlphaEvolve как продвинутый роутер) и Prod-ops (изолированные среды исполнения) (CP). Для SL2 AI Platform это давление на GAP G3 (observability) и G6 (LLM gateway), так как Vertex AI предлагает managed-решения с корпоративной поддержкой, в то время как SL2 AI Platform требует self-hosted альтернатив (SP). Однако уникальное преимущество SL2 AI Platform — git-native dev и независимое масштабирование — остаётся незыблемым.

Сигналы: AlphaEvolve GA, Cloud Run sandboxes

AWS Bedrock: AWS выпустил в GA Bedrock AgentCore Runtime с изоляцией агентов в отдельных microVM (DP), что закрывает критерии Indep. deploy/scale и Secrets/tenant (CP). Для SL2 AI Platform это прямое давление на GAP G3 (observability) и G4 (cost tracking), так как Bedrock теперь предлагает managed-решения с прозрачной изоляцией и потенциальным cost breakdown (SP). Однако Bedrock остаётся closed SaaS, что сохраняет преимущество SL2 AI Platform для регулируемых клиентов.

Сигналы: AgentCore GA, Claude apps gateway

Azure Foundry: Microsoft объявил о GA Claude в Foundry и анонсировал Agent Optimizer в private preview (DP). Это усиливает критерий Multi-tenancy через AI Trust Architecture (CP), но детали ограничены. Для SL2 AI Platform это давление на GAP G7 (зрелость), так как Foundry активно расширяет экосистему агентов (SP). Однако отсутствие self-hosted возможностей сохраняет нишу для SL2 AI Platform.

Сигналы: Claude GA in Foundry

AutoGen: В обсуждениях на GitHub появился RFD SR-OauthDelegation-v1.7 для безопасной делегированной работы агентов (DP). Это затрагивает критерии Unified API + контракт и Secrets/tenant (CP), так как предлагает стандартизированный подход к авторизации. Для SL2 AI Platform это давление на GAP G2 (Agent Protocol lock-in), так как AutoGen движется к совместимости с A2A-стандартом (SP).

Сигналы: SR-OauthDelegation-v1.7

AgentForce: Salesforce анонсировал AI Trust Architecture для Agent-to-Agent коммуникаций (DP), что усиливает критерии Multi-tenancy и Unified API + контракт (CP). Для SL2 AI Platform это давление на GAP G2 (Agent Protocol) и G7 (зрелость), так как AgentForce предлагает корпоративную архитектуру доверия (SP). Однако отсутствие self-hosted возможностей ограничивает конкуренцию с SL2 AI Platform.

Сигналы: AI Trust Architecture

LangGraph: Релизы 1.2.8/1.2.9 с улучшениями стабильности и CLI-инструментов (DP) не затрагивают напрямую критерии SL2 AI Platform (CP). Однако для SL2 AI Platform это сигнал о развитии конкурентной open-source платформы, что может повлиять на GAP G7 (зрелость) в долгосрочной перспективе (SP).

Сигналы: langgraph==1.2.9

n8n: Публикация статей о контекстной инженерии и AI-агентной памяти (DP) усиливает критерий Code+No-code через визуальные билдеры (CP). Для SL2 AI Platform это давление на GAP G1 (visual builder), так как n8n предлагает более низкий порог входа для не-разработчиков (SP).

Сигналы: Context Engineering, AI Agent Memory

NetBox: Обсуждения моделирования сложных сетевых топологий (vPC, splitter) (DP) не затрагивают напрямую критерии SL2 AI Platform (CP). Однако для SL2 AI Platform это сигнал о развитии инструментов для инфраструктурного моделирования, что может быть полезно для интеграции с агентными платформами (SP).

Сигналы: vPC Modeling

Snipe-IT: Обсуждение печати QR-кодов для аксессуаров (DP) не затрагивает критерии SL2 AI Platform (CP). Однако для SL2 AI Platform это сигнал о развитии инструментов для asset management, что может быть полезно для интеграции с агентными платформами (SP).

Сигналы: QR Code Printing
