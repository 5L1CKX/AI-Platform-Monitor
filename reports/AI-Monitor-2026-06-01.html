3. Детали по платформам

Vertex AI:

Google продолжает укреплять экосистему вокруг Gemini Enterprise, фокусируясь на защите данных и глубокой интеграции с BigQuery. Основное внимание уделено инструментам для разработчиков и защите критической инфраструктуры.

Сигналы: Интеграция Gemini Enterprise с A2UI, AI-ready безопасность для госсектора

AWS Bedrock:

AWS агрессивно развивает инструменты observability и оценки качества агентов, делая ставку на стандартизацию через Model Context Protocol (MCP). Акцент смещается в сторону управления жизненным циклом и масштабируемости LLM-приложений.

Сигналы: Наблюдаемость LLM через SageMaker, Управление тестами в Bedrock AgentCore

Azure Foundry:

Microsoft Foundry представила важные обновления для оценки агентов, работающих на любой инфраструктуре. Фокус направлен на улучшение контроля затрат и локальное исполнение моделей.

Сигналы: Обновления Azure Foundry май 2026

AutoGen:

Сообщество активно обсуждает проблемы governance в мультиагентных системах, включая аудит действий, аутентификацию агентов и безопасность tool-call. Разработчики ищут стандарты для production-ready координации.

Сигналы: Governance и security для tool-exec, Аутентификация агентов

AgentForce:

Salesforce активно продвигает использование MCP-серверов для внешних систем, стремясь сделать свою платформу центром управления для ИИ-агентов. Важным трендом является создание «headless» ИИ-интерфейсов.

Сигналы: Интеграция Claude с Salesforce MCP, Headless AI опыт

LangGraph:

Платформа демонстрирует стабильный цикл релизов, продолжая улучшать CLI и SDK для управления состоянием сложных агентов. Совместимость с новыми моделями расширяется через интеграционные пакеты.

Сигналы: Релиз LangGraph SDK 0.4.0, Интеграция Perplexity и Anthropic

n8n:

Платформа делает ставку на Sovereign AI, подчеркивая возможности self-hosted решений для строгого соблюдения residency-требований. Внедряются более строгие механизмы контроля действий агентов.

Сигналы: Sovereign AI и автоматизация, Контроль действий агентов

5. Выводы для команды

  1. Стандартизация через MCP становится отраслевым мейнстримом; SL2 AI Platform должна обеспечить нативную поддержку или полную совместимость с протоколом MCP в следующем релизе.
  2. Enterprise-клиенты всё чаще требуют «Governance as Code» — необходимо усилить функционал SL2 AI Platform в части аудита инструментов и разграничения прав доступа агентов.
  3. Sovereign AI и self-hosted инсталляции остаются нашим главным преимуществом против облачных провайдеров; нужно подчеркивать в маркетинге, что SL2 AI Platform дает полный контроль над данными, который Azure/AWS не могут гарантировать в полной мере.
  4. В фокус разработки SL2 AI Platform стоит добавить встроенные инструменты «offline evaluation», так как конкуренты активно закрывают этот пробел.
